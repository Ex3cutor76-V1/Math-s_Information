Segurança com o terminal:

Bem aqui vou te mostrar alguns comandos para a sua segurança no linux.

clamscan:

Bem além desse comando ser muito importante ele é ótimo para se caso no Linux você se sente com medo de pegar vírus, já que o clamscan na verdade é um antivírus da ClamAV que é um antivírus opensource que pode ser usado via terminal.

Como instalar:

Só usar o comando: sudo apt install clamav

Como atualizar:

Sim até eu fico impressionado disso, mas para atualizar ele é bem diferenciador já que você usa esse comando: sudo freshclam

iftop:

O comando iftop é semelhante ao top e o btop, a única diferença? É que ele serve para monitorar o tráfego de rede em tempo real e sim ele só consegue ser usado se você for root (Para entrar como root no terminal só usar sudo su)

E bem com esse comando durante o monitoramento em tempo real do tráfego de rede ele vai mostrar coisas como: IP, portas, taxa de transferência, bytes enviados e recebidos

Para instalar use: sudo apt-get install iftop

Como mexer no firewall pelo terminal Linux?

Bem quando nós queremos segurança a primeira coisa que pensamos é no firewall, já que é o firewall que permite o que entra e o que sai do sistema baseado em regras e bem é possível mexer no firewall através do terminal e claro só irei mostrar os comandos do firewall que são mais usados, já que tem alguns que não irão ser tão úteis, já que são bem específicos.

sudo ufw status:

Com este ele só avisa se o seu firewall está ativo ou não.

sudo ufw enable:

Com este comando você ativa o firewall

sudo ufw disable:

Com este comando você desativa o firewall

sudo ufw allow (porta):

Com este comando você permite uma porta em específica, abaixo estará um exemplo:

Na imagem de exemplo eu acabei de permitir a porta 80.

sudo ufw deny (Porta):

Com este comando você bloqueia uma porta em específica, abaixo estará uma imagem de exemplo.

Na imagem de exemplo eu acabei de bloquear a porta 80.

sudo ufw delete allow (porta):

Com este comando você remove uma regra, abaixo estará um exemplo:

Na imagem por exemplo, eu acabei de deletar a regra da porta 80.

Alguns comandos bem úteis também:

sudo ufw status numbered: Com este comando você consegue ver algumas regras existentes no seu firewall.

sudo ufw limit ssh: Limita tentativas de conexão (Perfeita para se proteger de ataques de DDoS).

sudo ufw status verbose: Ele é tipo o "sudo ufw status" só que ele te dá uma visão mais avançada sobre seu firewall.

Atualização do dia 28/01/2026:

Olá caro visitante, você provavelmente lembra dos comandos do firewall certo? Mas também deve ter se perguntado qual porta bloquear, já que algumas delas são sim inseguras e bem por mais que não é só porta TCP/IP que realmente pode te colocar em risco mas sim qual programa está escutando deixa eu aliviar um pouco sua parte por porta TCP/IP porque elas são bem antigas e inseguras.

Portas TCP/IP inseguras: 21, 23, 69, 137, 139, 445, 512, 514, 111.

Explicação de cada porta:

21: FTP (Envio e download de arquivos mas não é criptografado.)

23: Telnet (Acesso remoto a terminal, mas tudo é texto puro e foi substituido pelo SSH que é a porta 22.)

69: TFTP (Transferência simples de arquivos e isso sem login, no caso sem autênticação e sem criptografia.)

111: RPCbind (Descobre serviços RPC em sistemas Unix/Linux e sim ele é um alvo comum a ataques se exposto à internet, mas normalmente é usado em redes internas.)

137: NetBIOS Name Service (Resolução de nomes em redes windows antigas que bem pode vazar nomes de máquinas e usuários.)

139: NetBIOS Session Service (Compartilhamento de arquivos e impressoras no windows antigo e sim é inseguroe exposto externamente.)

445: SMB (Compartilhamento de arquivos e impressoras no windows mas é uma porta famosa por ataques, um exemplo disso é o ataque WannaCry.)

512: rexec (Execução remota de comandos, usado como um serviço Unix antigo e sim é sem criptografia.)

514: rsh /syslog (rsh é o shell remoto, já o syslog é um log do sistema UDP, mas o rsh é inseguro, syslog é até seguro mas, como possui rsh não parece ser uma boa ideia deixar aberto, mas isso é por sua conta e risco.)

Bem certo até te apresentei algumas portas que realmente devem ser fechadas, mas lembra quando eu disse que o problema não é só a porta TCP/IP como também é quem está escutando? No caso o seu programa que está escutando, ele é o real problema e bem esse aqui é o comando para verificar isso:

ss -tulpet

Esse comando muitas vezes é usado para verificar quem está usando qual porta e sim pode ser meio bagunçado, mas se ler com atenção talvez isso te ajude.
