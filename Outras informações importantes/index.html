Informações importantes para você entender o que é:

Bem aqui neste tópico será mostrado algumas informações básicas que você provavelmente vai ver no seu dia-a-dia como hacker ético.

IP (Internet Protocol/Protocolo de Internet): é basicamente um endereço exclusivo que identifica um dispositivo na internet ou em uma rede local.

Estrutura de um endereço IP: n.n.n.n (n = Octeto, no caso um conjunto de 8 bits.)

MAC (Media Access Control/Controle de Acesso à Mídia) Adress: É um identificador único de 12 digitos, composto por números e letras hexadecimais que é atribuido pelo fabricante a cada componente de hardware de uma rede, resumidamente é tipo um CPF para o computador.

Estrutura de um MAC Adress: São de 48 bits e compostos por 6 bytes e é expresso de forma concisa em notação hexadecimal, separado como dois pontos (:) ou hifens (-) para melhor legibilidade. Exemplo: 00:0A:95:9D:67:16, onde "00:0A:95" é o OUI, que é o indicador da placa de rede e sim essa parte fala qual é a o fabricante, já o "9D:67:16" é o identificador exclusivo atribuído pelo fabricante para a placa de rede específica.

TCP/IP: É um número usado pelos computadores para identificar serviços ou aplicações específicas que estão se comunicando pela rede.

Estrutura de uma porta TCP: É um número de 16 bits no caso possui portas de 0 a 65535.

Portas UDP(User Datagram Protocol/Protocolo datagrama de usuário): É um ponto de comunicação assim como o TCP, a única diferença é que é sem conexão e é mais rápido que o TCP, mas não é tão confiável, pois, não garante entrega dos pacotes.

Estrutura da porta UDP: Simplesmente um tamanho de 8 bytes e um campo de dados (payloads).

HTTP (Hypertext Transfer Protocol/Protocolo de Transferência de Hipertexto): É a base web e um conjunto de regras que permite o navegador do seu computador solicitar e receber informações de um servidor web e vice-versa.

Estrutura de um HTTP: http://www.nomedosite.com

HTTPS (Hypertext Transfer Protocol Secure/Protocolo de Transferência de Hipertexto Seguro): É a versão segura do HTTP, até porque ele utiliza criptografia.

Estrutura de um HTTPS: https://www.nomedosite.com

FTP (File Transfer Protocol/Protocolo de Transferência de Arquivos): É um protocolo de rede que usa um conjunto de regras para transferir arquivos entre computadores de uma rede TCP/IP.

Estrutura de um FTP: ftp://utilizador:password@host:porta/path

SSH (Secure Shell): É um protocolo de rede que permite acesso remoto e seguro aos computadores utilizando criptografia para proteger dados.

Estrutura de uma SSH: Ela se baseia num par de chaves criptográficas no caso uma chave privada e uma chave pública.

SMTP (Simple Mail Transfer Protocol/Protocolo de Transferência de Email Simples): É literalmente um protocolo padrão de enviar um email ou receber um email pela internet.

Estrutura de um SMTP: Envolve um cliente e um servidor que se comunicam através de comandos de textos para envio de email online.

DNS (Domain Name's of System/Nome de Sistemas de Domínio): É basicamente a "lista telefônica" da internet, pois, traduz nomes de domínios legíveis por humanos.

Estrutura do DNS: www.nomedosite.com

CDN (Content Delivery Network): É uma rede de servidores distribuídos pelo mundo que armazena cópias de conteúdos de sites (Como imagens, vídeos e etc) para entregar de forma mais rápida aos usuários.

Exemplo de CDN: Cloudflare

ASN (Autonomous System Number): É um identificador único usado na internet para representar uma rede independente que administra seu próprio tráfego IP.

Firewall: É uma barreira de segurança que controla e monitora o tráfego de rede considerando o que entra e o que sai com base em regras predefinidas.

Plugins: Extensões de softwares ou módulos adicionais para expandir suas funcionalidades sem a necessidade de alterar o código do site diretamente.

Por enquanto é só isso que deve saber, já que você irá utilizar isso no seu dia-a-dia como hacker ético, então... Aproveite!
